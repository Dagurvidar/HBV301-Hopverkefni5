# Verkefni 5 — Gæðaeiginleikar (HBV301G)

## Nafn kerfis  
**Nemendakerfi fyrir námsmat**

---

## Stutt lýsing á kerfi  
Nemendakerfi fyrir námsmat er hannað til að einfalda og bæta ferlið við verkefnaskil, einkunnagjöf og yfirsýn námsárangurs í menntastofnunum.  
Kerfið þjónar bæði nemendum og kennurum: nemendur geta skilað verkefnum og fylgst með einkunnum sínum, en kennarar geta fært inn einkunnir, gefið endurgjöf og fylgst með framvindu hópa.  
Helstu markmið kerfisins eru að tryggja **gagnsæi, áreiðanleika og einfaldleika í námsmati**, auk þess að styðja örugga og aðgengilega notendaupplifun í skýjagrunduðu umhverfi.

## Yfirlit  

Í þessu verkefni unnu Dagur og Stefán að því að greina, skilgreina og skjalfesta gæðaeiginleika fyrir **Nemendakerfi fyrir námsmat**.  
Markmiðið var að tryggja að bæði innri og ytri gæði kerfisins væru mælanleg og rekjanleg, í samræmi við leiðbeiningar úr *Wiegers & Beatty (kafli 14)*.  

Stefán vann með **fimm ytri gæðaeiginleika** sem snúa að notendaupplifun og rekstri — notagildi, frammistöðu, öryggi, aðgengi og áreiðanleika.  
Dagur vann með **þrjá innri gæðaeiginleika**, sem tengjast þróun og viðhaldi kerfisins, og skrifaði eina kröfu í **formlegu PLanguage-sniði**.  

Saman skilgreindu þeir **þrjú árekstrapör**, þar sem mismunandi gæðaeiginleikar gætu rekist á, t.d. *öryggi vs. aðgengi* og *frammistaða vs. áreiðanleiki*.  
Þeir rökstuddu forgangsröðun byggða á viðskiptamarkmiðum, áhættu og notendaupplifun, með áherslu á að tryggja jafnvægi milli tæknilegra og mannlegra þátta.

## Ferli fyrir verkefnið  

1. Hópurinn byrjaði á því að velja gæðaeiginleika út frá fyrri verkefnum (1–3) og tengja þá við markmið og notendakröfur kerfisins.  
2. Dagur og Stefán skiptu verkefninu upp: Dagur vann með innri eiginleika og PLanguage, Stefán með ytri eiginleika og árekstrapar.  
3. Hver gæðaeiginleiki var skráður samkvæmt `QUALITY_ATTRIBUTE.md` sniðmáti, með mælanlegri kröfu og rökstuðningi.  
4. Hópurinn skilgreindi þrjú árekstrapör og forgangsröðun þeirra samkvæmt `CONFLICT_PAIR.md`.  
5. Verkefnið var unnið sameiginlega í GitHub með reglulegum commitum, pull requestum og kóðaskoðun (peer review).
